Encadrepro agent prompt 02 ui propagation · MDCopyEncadrePro — Agent Task 02: Propagate 5-Type Product System Through All UI
Context
In the previous task (commit 65adcb8), the database schema was migrated from 3 product types (Matière / Façonnage / Service) to 5 (VR / FA / AU / SD / EN), per the legacy FileMaker app the client uses. The API server, OpenAPI spec, generated Zod schemas, and React Query hooks all now know about the 5 types.
The UI was deliberately not rebuilt in that pass — it was scoped to schema only. As a result, the user-facing pages still reflect the old 3-type model and the catalog appears empty (test data was wiped during migration).
This task propagates the 5-type system through every UI surface: catalog browser, product detail/edit forms, the devis (quote) composer, line item entry, search filters, and anywhere else the old 3-type list is hardcoded. After this task, a user should be able to navigate the full app and create a devis containing line items of all 5 product types.
Scope discipline reminder: This is a UI propagation task, not a feature build. You are replacing references to the old 3-type system with the new 5-type system, plus adding the minimum type-conditional UI required for each type's specific fields. You are NOT building new screens, new workflows, or new components beyond what's required to expose the 5 types correctly.
