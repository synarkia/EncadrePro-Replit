Think of yourself as a QA engineer, not a developer, for this pass.

## Hard constraints

1. **Do not build new features.** If a screen is missing (e.g. suppliers list page), report it — don't create it.
2. **Small fixes only.** A typo, a missing `res.ok` check, a wrong badge color — fix it. A refactor, a new component, a schema change — report it.
3. **No invented test results.** If you can't reach a page, write BLOCKED with the error. If a price computation doesn't match the expected value, write FAIL with both numbers. Do not round to "close enough."
4. **No new dependencies.**
5. **French UI.** If you find English text anywhere it shouldn't be, flag it.
6. **Commit seeds and any bug fixes separately** so the diff is reviewable. Don't bundle test fixes with seed data.
