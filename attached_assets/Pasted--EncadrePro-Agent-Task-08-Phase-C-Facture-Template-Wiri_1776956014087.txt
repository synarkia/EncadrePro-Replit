## Context

A devis PDF renderer is already working (proof: the screenshot with DEV-2026-003 for Ryan O'CALLAGHAN-LOCKE rendered with the Atelier AGV design system — wordmark, laiton rule, serif/sans hierarchy, nested line sub-items, "Bon pour accord" signature zone, TVA art. 293 B footer). This task does three things:

1. **Brings the facture renderer to full parity** with the `Facture_Atelier_AGV.html` template the user crafted (multi-rate TVA, acompte versé / net à payer block, cachet & signature, bank details, expanded legal footer).

- `lignes_devis.description_longue text` (nullable)

### Line rendering details (the heart of Option B)

Your template's line cell structure is:

```html
<td class="lines__desc">
  <strong>[Product name / title]</strong>
  <small>[Rich description, dimensions, sub-items]</small>
</td>
```

The `<small>` section should be built by concatenating, in order, only the parts that have values:

1. If `description_longue` is non-empty: its content (may be multi-line — wrap `\n` as `<br>`)
2. If dimensions are set: append a line like `Format 280×0 cm` (for linear) or `Format 56,5×71 cm` (for surface) or nothing (for unit)
3. For each façonnage sub-item of this line: `{designation} — {formatEuro(total_ht)}` on its own line
4. For each service sub-item of this line: `{designation} — {formatEuro(total_ht)}` on its own line

Match the screenshot's existing pattern — the devis renderer already does this well for sub-items. Verify the existing code handles `description_longue` being present (new) and falls back gracefully when null.

## Step 4 — PDF export + print flow

### "Télécharger PDF" button

On `pages/devis/[id].tsx` and `pages/factures/[id].tsx`, add a button "Télécharger PDF" near the existing action buttons. Clicking it:

1. Calls `GET /api/devis/:id/pdf` or `GET /api/factures/:id/pdf`
2. Server side:
   - Renders the HTML template with the mapped data
   - Uses Puppeteer (install if not already) to convert HTML → A4 PDF with `printBackground: true`, margins `0` (the `.sheet` class handles margins via CSS), format `A4`
   - Returns PDF as download with filename `Devis-DEV-2026-003-OCallaghan-Locke.pdf` (sanitize special chars for filename)
3. Browser downloads the PDF

### "Imprimer" button

Opens the rendered HTML in a new tab at `/devis/:id/print` (and `/factures/:id/print`). The print route serves the HTML standalone (no app chrome) with `<script>window.print()</script>` auto-triggering the print dialog. The existing `@media print` rules in the template handle A4 layout.

### Dependencies

- Install `puppeteer` (not puppeteer-core — we want the bundled Chromium for simplicity on Replit). Size warning: Chromium is ~170MB. If that's prohibitive on Replit's deployment, fall back to `@sparticuz/chromium-min` + `puppeteer-core` — flag the choice in the plan.
- If Puppeteer already exists for some other purpose, reuse it.

### Security

- PDF generation routes require the user to have access to the devis/facture being rendered (not a concern yet since no auth exists, but structure the code so auth can drop in without refactor)
- Sanitize all `data-var` values before injection to prevent HTML injection from malicious client names (escape `<`, `>`, `&`, `"`, `'`)
- Do not log PDF contents in server logs — may contain PII

## Hard constraints

1. **No multi-rate TVA implementation** in this task. Keep single-rate. Template shows em-dash placeholders for 10% and 5.5% rows.
2. **No auth layer changes** — out of scope.
3. **No refactoring unrelated to template mapping, description_longue, and PDF export.**

5. **PDF filename sanitization** — strip accents (NFD normalization + stripping combining marks), replace spaces with `-`, keep only `[A-Za-z0-9-_]`.
6. **Template file lives in the repo** — do not fetch HTML from an external URL at request time. Copy `Facture_Atelier_AGV.html` into `lib/documents/templates/` and bundle with the server.
7. **If the existing devis renderer already uses a templating approach** (Handlebars, etc.), continue with that approach rather than switching libraries. If it uses raw string replacement, fine — keep it if it works; the template has explicit `data-var` attributes that are trivial to substitute.
8. **Money math identical to on-screen totals.** The PDF must never show a number different from what the UI shows. Reuse the same `computeLigneTotalHT` function that drives the live preview.
9. **No new schema columns beyond what's listed** in Steps 1 and the atelier extensions in Step 2. If another column feels "necessary," flag it in your orientation, don't add it silently.
